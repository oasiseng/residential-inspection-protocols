# Protocol Template (Residential Existing Homes)

## 1) Scope
- System and boundaries covered.
- Explicitly note what is **out of scope**.

## 2) Objective
- What decision this protocol supports (Pass / Monitor / Escalate).

## 3) Code & Standard Basis
- **Code basis:** IEBC/IRC/IBC context for existing conditions.
- **Standard basis:** ASTM/ISO/ASHRAE references used for method structure or interpretation.
- Clarify whether each citation is **required** or **guidance-only**.

## 4) Tools Required
- Include minimum acceptable instrument class.
- Include calibration/verification note where relevant.

## 5) Safety + Non-Destructive Limits
- Explicitly state prohibited invasive/destructive actions.
- Include stop conditions tied to occupant/property risk.

## 6) Preconditions
- Environmental, occupancy, and system runtime assumptions.

## 7) Procedure Steps
- Stepwise field sequence.
- Include checkpoints and observable criteria.

## 8) Interpretation Criteria
- Define **Pass / Monitor / Escalate** thresholds.
- Include ambiguity handling (what to do when data conflicts).

## 9) What AI Gets Wrong
- At least 3 common bad recommendations specific to the discipline.
- Include one example of a high-risk hallucinated action.

## 10) Documentation Minimum
- Required photos, measurements, instrument IDs, and field notes.
- Required language for limitations and next actions.

## 11) Escalation Path
- When to involve specialty contractor, licensed PE, or emergency response.
